---
title: "SecretProviderClass parameters and descriptors reference"
description: "Complete reference for all parameters and object descriptor fields in the SecretProviderClass CRD used by the AWS Secrets Store CSI Driver Provider."
---

The `SecretProviderClass` custom resource tells the AWS provider which secrets and parameters to fetch and how to mount them inside your pods. You create one per set of secrets your application needs, and reference it from a `CSIDriver` volume in your pod spec.

Every `SecretProviderClass` for the AWS provider follows this skeleton:

```yaml
apiVersion: secrets-store.csi.x-k8s.io/v1
kind: SecretProviderClass
metadata:
  name: my-app-secrets
spec:
  provider: aws
  parameters:
    region: us-east-1
    objects: |
      - objectName: "MySecret"
        objectType: "secretsmanager"
```

---

## Top-level parameters

These fields live directly under `spec.parameters`.

<ParamField path="objects" type="string" required>
  A YAML-formatted string that lists every secret or parameter to mount. Write it as a multi-line YAML string using the pipe (`|`) block scalar. Each entry in the list is an [object descriptor](#object-descriptor-fields) and may reference a Secrets Manager secret, an SSM Parameter Store parameter, or both.

  ```yaml
  parameters:
    objects: |
      - objectName: "MySecret"
        objectType: "secretsmanager"
      - objectName: "/my-app/db-password"
        objectType: "ssmparameter"
  ```
</ParamField>

<ParamField path="region" type="string">
  The AWS region from which secrets are retrieved. When omitted, the provider resolves the region by reading the `topology.kubernetes.io/region` label on the node running the pod. Providing this value explicitly avoids that extra Kubernetes API call, which improves mount latency on clusters that mount large numbers of pods simultaneously.

  ```yaml
  parameters:
    region: us-west-2
  ```
</ParamField>

<ParamField path="failoverRegion" type="string">
  A secondary AWS region used when the primary region is unavailable. The provider attempts to fetch secrets from both regions in parallel:

  - If both succeed, the primary region's value is used.
  - If one region returns a server-side error (5xx), the value from the healthy region is used.
  - If either region returns a client error (4xx), the mount fails and must be corrected before retrying.

  `failoverRegion` must differ from `region`. When this field is present, individual object descriptors may include a [`failoverObject`](#failoverobject) entry to specify a different secret name or ARN in the secondary region.

  ```yaml
  parameters:
    region: us-east-1
    failoverRegion: us-east-2
  ```
</ParamField>

<ParamField path="pathTranslation" type="string" default="_">
  A single substitution character used to replace the path separator (`/`) in secret names when they are written as file names. SSM Parameter Store names frequently contain slashes (e.g. `/my-app/db/password`), and those cannot map directly to a single file name.

  - **Default** (`_`): `My/Path/Secret` is mounted as `My_Path_Secret`.
  - **`"False"`**: Disables substitution entirely. Use `objectAlias` to supply a safe file name when translation is off.
  - **Any single character**: Uses that character as the replacement (e.g. `"-"` mounts `My/Path/Secret` as `My-Path-Secret`).

  See [Configure path translation](/configuration/path-translation) for full details and examples.
</ParamField>

<ParamField path="usePodIdentity" type="string">
  Selects the IAM authentication method. When not set, the provider defaults to IAM Roles for Service Accounts (IRSA).

  | Value | Auth method |
  |---|---|
  | `"true"`, `"True"`, `"TRUE"`, `"t"`, `"T"` | EKS Pod Identity |
  | `"false"`, `"False"`, `"FALSE"`, `"f"`, `"F"` | IRSA (explicit) |
  | *(omitted)* | IRSA (default) |

  EKS Pod Identity requires EKS 1.24 or later and the EKS Pod Identity Agent add-on installed on the cluster.

  ```yaml
  parameters:
    usePodIdentity: "true"
  ```
</ParamField>

<ParamField path="preferredAddressType" type="string">
  Controls which IP address family is used to reach the Pod Identity Agent endpoint. This field is ignored when `usePodIdentity` is not enabled.

  | Value | Behavior |
  |---|---|
  | `"ipv4"`, `"IPv4"`, `"IPV4"` | Force IPv4 endpoint |
  | `"ipv6"`, `"IPv6"`, `"IPV6"` | Force IPv6 endpoint |
  | *(omitted)* | Auto-select: try IPv4, fall back to IPv6 |

  Values are case-insensitive.
</ParamField>

## Object descriptor fields

Each entry in the `objects` list is an object descriptor. These fields configure exactly which secret to fetch and how to present it as a file in the pod.

<ParamField path="objectName" type="string" required>
  The identifier of the secret or parameter to fetch.

  - **Secrets Manager**: accepts the friendly name (e.g. `"MySecret"`) or the full ARN (e.g. `"arn:aws:secretsmanager:us-east-1:123456789012:secret:MySecret-AbCdEf"`).
  - **SSM Parameter Store**: accepts the parameter name (e.g. `"/my-app/db/password"`) or the full ARN.

  When a full Secrets Manager ARN is provided, `objectType` is optional because the type can be inferred from the ARN. For SSM Parameter Store, `objectType` must always be set explicitly unless a full ARN is used and the ARN contains the SSM service identifier.
</ParamField>

<ParamField path="objectType" type="string">
  The AWS service storing the secret. Required unless `objectName` is a full Secrets Manager ARN.

  | Value | Service |
  |---|---|
  | `"secretsmanager"` | AWS Secrets Manager |
  | `"ssmparameter"` | AWS Systems Manager Parameter Store |
</ParamField>

<ParamField path="objectAlias" type="string">
  The file name under which the secret is written in the mount directory. When omitted, the file is named after `objectName` (with path translation applied). Use `objectAlias` to give the file a stable, application-friendly name regardless of the secret's name in AWS.

  `objectAlias` is **required** when a `failoverObject` is present.

  ```yaml
  - objectName: "arn:aws:secretsmanager:us-east-1:123456789012:secret:PrimarySecret-12345"
    objectAlias: "db-password"
    failoverObject:
      objectName: "arn:aws:secretsmanager:us-east-2:123456789012:secret:FailoverSecret-12345"
  ```
</ParamField>

<ParamField path="objectVersion" type="string">
  Pins the mount to a specific version of the secret. When omitted, the latest version is used.

  - **Secrets Manager**: maps to the `VersionId` parameter.
  - **SSM Parameter Store**: maps to the numeric version number.

  <Warning>
    Specifying `objectVersion` requires manual updates to this field whenever you rotate the secret. Omitting it is recommended unless you need strict version pinning.
  </Warning>

  SSM Parameter Store does not allow specifying both `objectVersion` and `objectVersionLabel` for the same object.
</ParamField>

<ParamField path="objectVersionLabel" type="string">
  Pins the mount to a named version stage or label. Most applications should not set this field because the current version is used by default.

  - **Secrets Manager**: maps to the `VersionStage` parameter (e.g. `"AWSCURRENT"`, `"AWSPREVIOUS"`).
  - **SSM Parameter Store**: maps to a Parameter Label.

  SSM Parameter Store does not allow specifying both `objectVersion` and `objectVersionLabel` for the same object.
</ParamField>

<ParamField path="filePermission" type="string" default="0644">
  A four-digit octal string that sets the Unix file permissions on the mounted secret file. The value must be a valid four-digit octal string (e.g. `"0400"`, `"0640"`).

  ```yaml
  - objectName: "MySecret"
    objectType: "secretsmanager"
    filePermission: "0400"
  ```
</ParamField>

<ParamField path="jmesPath" type="array">
  Extracts individual key-value pairs from a JSON-formatted secret value and mounts each one as a separate file. This is useful when a single Secrets Manager secret stores multiple credentials as a JSON object.

  Each entry in `jmesPath` requires two fields and accepts one optional field:

  <ParamField path="jmesPath[].path" type="string" required>
    The [JMESPath](https://jmespath.org/specification.html) expression identifying the key to extract from the JSON secret value. If the path contains a hyphen, wrap it in single quotes: `'"hyphenated-key"'`.
  </ParamField>

  <ParamField path="jmesPath[].objectAlias" type="string" required>
    The file name under which the extracted value is mounted. If the alias contains a hyphen, wrap it in single quotes: `'"hyphenated-alias"'`.
  </ParamField>

  <ParamField path="jmesPath[].filePermission" type="string">
    Per-entry file permission override. Defaults to the parent object's `filePermission` when omitted.
  </ParamField>

  ```yaml
  - objectName: "MySecret"
    objectType: "secretsmanager"
    jmesPath:
      - path: "username"
        objectAlias: "db-username"
      - path: "password"
        objectAlias: "db-password"
        filePermission: "0400"
  ```
</ParamField>

<ParamField path="failoverObject" type="object">
  Specifies an alternative secret to fetch from the `failoverRegion` when the primary region is unavailable. Requires `failoverRegion` to be defined in `spec.parameters` and `objectAlias` to be set on the parent descriptor.

  <ParamField path="failoverObject.objectName" type="string" required>
    The name or ARN of the secret in the failover region. If an ARN is used, its region segment must match `failoverRegion`. Required when `failoverObject` is present.
  </ParamField>

  <ParamField path="failoverObject.objectVersion" type="string">
    Version ID for the failover secret. When specified, it must match the primary object's `objectVersion`.
  </ParamField>

  <ParamField path="failoverObject.objectVersionLabel" type="string">
    Version label for the failover secret. See the primary `objectVersionLabel` field for details.
  </ParamField>

  ```yaml
  parameters:
    region: us-east-1
    failoverRegion: us-east-2
    objects: |
      - objectName: "arn:aws:secretsmanager:us-east-1:123456789012:secret:PrimarySecret-12345"
        objectAlias: api-key
        failoverObject:
          objectName: "arn:aws:secretsmanager:us-east-2:123456789012:secret:FailoverSecret-12345"
  ```
</ParamField>

## Complete example

The following `SecretProviderClass` demonstrates all major features: a plain Secrets Manager secret, a JSON secret with JMESPath extraction, an SSM parameter, and a failover configuration.

```yaml
apiVersion: secrets-store.csi.x-k8s.io/v1
kind: SecretProviderClass
metadata:
  name: my-app-secrets
  namespace: default
spec:
  provider: aws
  parameters:
    region: us-east-1
    failoverRegion: us-east-2
    pathTranslation: "_"
    usePodIdentity: "false"

    objects: |
      # Plain Secrets Manager secret
      - objectName: "MyApiKey"
        objectType: "secretsmanager"
        filePermission: "0400"

      # JSON secret with JMESPath extraction
      - objectName: "MyDatabaseCredentials"
        objectType: "secretsmanager"
        jmesPath:
          - path: "username"
            objectAlias: "db-username"
          - path: "password"
            objectAlias: "db-password"
            filePermission: "0400"

      # SSM Parameter Store parameter
      - objectName: "/my-app/config/log-level"
        objectType: "ssmparameter"
        objectAlias: "log-level"

      # Secret with automated failover
      - objectName: "arn:aws:secretsmanager:us-east-1:123456789012:secret:PrimarySecret-AbCdEf"
        objectAlias: "primary-secret"
        failoverObject:
          objectName: "arn:aws:secretsmanager:us-east-2:123456789012:secret:FailoverSecret-XyZaBc"
```

<Note>
  The `objects` value is a YAML string embedded inside the `SecretProviderClass` YAML. The inner content is indented relative to the `objects:` key and uses the `|` block scalar so that whitespace and newlines are preserved correctly.
</Note>
