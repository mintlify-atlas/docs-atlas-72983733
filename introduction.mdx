The AWS Secrets Store CSI Driver Provider (ASCP) integrates AWS secret storage services with Kubernetes, letting you securely mount secrets from AWS Secrets Manager and AWS Systems Manager Parameter Store into your pods as files — without writing secrets into Kubernetes objects or your container images.

## What is the AWS Secrets Store CSI Driver Provider?

Many applications expect secrets to be available as files on disk. The ASCP bridges AWS secret storage services and Kubernetes by acting as a provider plugin for the [Secrets Store CSI Driver](https://secrets-store-csi-driver.sigs.k8s.io/). When a pod is scheduled, the driver calls this provider, which fetches your secrets from AWS and presents them as mounted files inside the pod.

Key benefits:
- **No secrets in Kubernetes**: Secrets never get stored in etcd or Kubernetes `Secret` objects (unless you configure syncing).
- **Fine-grained IAM access**: Each pod uses its own IAM role, scoped to only the secrets it needs.
- **Automatic rotation support**: Works with the CSI driver's rotation reconciler to keep mounted secrets up to date.
- **Multi-region failover**: Configure a secondary AWS region to maintain availability during outages.
