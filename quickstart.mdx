---
title: "Quickstart: mount your first AWS secret in Kubernetes"
description: "Create an AWS secret, configure IAM permissions with IRSA, define a SecretProviderClass, and mount the secret into an nginx pod on EKS."
---

This guide walks you through mounting an AWS Secrets Manager secret into a Kubernetes pod using IAM Roles for Service Accounts (IRSA). By the end, you will have a running nginx deployment with a secret available as a file at `/mnt/secrets-store/MySecret`.

Before you begin, make sure the ASCP is installed on your cluster. See [Install the AWS Secrets Store CSI Driver Provider](/installation) if you have not done so.

<Steps>
  <Step title="Set your region and cluster name">
    Export your AWS region and EKS cluster name as shell variables. These are used in subsequent commands.

    ```bash
    REGION=<REGION>
    CLUSTERNAME=<CLUSTERNAME>
    ```

    Replace `<REGION>` with the AWS region your cluster runs in (for example, `us-east-1`) and `<CLUSTERNAME>` with the name of your EKS cluster.
  </Step>

  <Step title="Create a test secret">
    Create a secret in AWS Secrets Manager with a username and password:

    ```bash
    aws --region "$REGION" secretsmanager create-secret \
      --name MySecret \
      --secret-string '{"username":"memeuser", "password":"hunter2"}'
    ```
  </Step>

  <Step title="Create an IAM policy scoped to the secret">
    Create an IAM policy that grants read access to `MySecret` only, and save its ARN to a variable:

    ```bash
    POLICY_ARN=$(aws --region "$REGION" --query Policy.Arn --output text iam create-policy \
      --policy-name nginx-deployment-policy \
      --policy-document '{
        "Version": "2012-10-17",
        "Statement": [ {
          "Effect": "Allow",
          "Action": ["secretsmanager:GetSecretValue", "secretsmanager:DescribeSecret"],
          "Resource": ["arn:*:secretsmanager:*:*:secret:MySecret-??????"]
        } ]
      }')
    ```

    <Note>
      If you are accessing SSM Parameter Store parameters, also include `"ssm:GetParameters"` in the `Action` list. You can tighten the `Resource` further by using the full secret ARN returned by `create-secret`.
    </Note>
  </Step>

  <Step title="Create the OIDC provider and IAM service account">
    First, associate an IAM OIDC provider with your cluster if you have not already done so:

    ```bash
    eksctl utils associate-iam-oidc-provider \
      --region="$REGION" \
      --cluster="$CLUSTERNAME" \
      --approve
    ```

    Then create the Kubernetes service account and attach the IAM policy to it using IRSA:

    ```bash
    eksctl create iamserviceaccount \
      --name nginx-irsa-deployment-sa \
      --region="$REGION" \
      --cluster "$CLUSTERNAME" \
      --attach-policy-arn "$POLICY_ARN" \
      --approve \
      --override-existing-serviceaccounts
    ```

    <Note>
      For private clusters, ensure the VPC has an AWS STS interface endpoint. See [Interface VPC endpoints](https://docs.aws.amazon.com/IAM/latest/UserGuide/reference_interface_vpc_endpoints.html) in the IAM User Guide for details.
    </Note>
  </Step>

  <Step title="Apply the SecretProviderClass">
    The `SecretProviderClass` tells the ASCP which secrets to fetch and mount. Apply the example that mounts `MySecret`:

    ```bash
    kubectl apply -f https://raw.githubusercontent.com/aws/secrets-store-csi-driver-provider-aws/main/examples/ExampleSecretProviderClass-IRSA.yaml
    ```

    This creates the following resource in your cluster:

    ```yaml
    apiVersion: secrets-store.csi.x-k8s.io/v1
    kind: SecretProviderClass
    metadata:
      name: nginx-irsa-deployment-aws-secrets
    spec:
      provider: aws
      parameters:
        objects: |
            - objectName: "MySecret"
              objectType: "secretsmanager"
    ```
  </Step>

  <Step title="Deploy the example nginx pod">
    Deploy the sample nginx application. It uses the service account created in the previous step and mounts the secret volume at `/mnt/secrets-store`:

    ```bash
    kubectl apply -f https://raw.githubusercontent.com/aws/secrets-store-csi-driver-provider-aws/main/examples/ExampleDeployment-IRSA.yaml
    ```

    Wait for the pods to reach `Running` status:

    ```bash
    kubectl get pods -l app=nginx-irsa
    ```
  </Step>

  <Step title="Verify the secret is mounted">
    Once the pod is running, read the mounted secret file to confirm the secret was fetched and mounted correctly:

    ```bash
    kubectl exec -it $(kubectl get pods | awk '/nginx-irsa-deployment/{print $1}' | head -1) \
      -- cat /mnt/secrets-store/MySecret; echo
    ```

    You should see the secret value printed to your terminal:

    ```json
    {"username":"memeuser", "password":"hunter2"}
    ```
  </Step>
</Steps>
